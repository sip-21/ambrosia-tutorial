# Managing Users

Learn how to add, edit, and manage staff members in your Ambrosia Point of Sale system.

## Overview

The Users section allows you to manage all staff members who will be using the POS system. Each user can have different roles and permissions, and uses a secure 4-digit PIN for quick access during shifts.

## Accessing the Users Section

1. From the Dashboard, click on **Users** in the left sidebar menu
2. You'll see a table displaying all current users with the following information:
   - Name
   - Role
   - Email
   - Phone
   - Actions (Edit and Delete buttons)

## Adding a New User

### Step 1: Open the Add User Dialog

Click the **Add User** button in the top-right corner of the Users page.

### Step 2: Fill in User Information

A modal dialog will appear with the following fields:

#### Name
- Enter the full name of the staff member
- Example: `Albert`, `Jane Smith`

#### Email
- Enter the user's email address
- Example: `albert@yourstore.com`

#### Phone
- Enter the user's phone number
- Format: International format recommended
- Example: `3312345678`

#### PIN (4-digit)
- Create a secure 4-digit PIN for the user
- This PIN will be used by the staff member to access the POS

#### Role
- Select the user's role from the dropdown menu

### Step 3: Save the User

Once all required fields are filled:

1. Review the information for accuracy
2. Click the **Add** button to create the user
3. Click **Cancel** if you want to discard the changes

The new user will immediately appear in the users table.

## Editing a User

To modify an existing user's information:

1. Locate the user in the users table
2. Click the blue **Edit** button (pencil icon) in the Actions column
3. The Edit User dialog will appear with the current information pre-filled
4. Make your changes
5. Click **Save** to update the user information

## Deleting a User

To remove a user from the system:

1. Locate the user in the users table
2. Click the red **Delete** button (trash icon) in the Actions column
3. Confirm the deletion when prompted
4. The user will be immediately removed from the system

:::danger
Deleting a user is permanent. Make sure you really want to remove the user before confirming.
:::
